The deep learning model that was trained on a 2013 dataset

Back in 2018, we worked on a deep learning model for video recommendations (suggesting TV shows and movies based on your prior watch history) that was initially evaluated on an old static dataset from 2013‬. This deep learning model was originally developed by a peer research team and was developed using a Hadoop cluster that resided on physical infrastructure.

‪Our first goal was to reproduce the evaluation #s (i.e. recall) by training and evaluating (offline) the model in our AWS account, with intentions of not only comparing how long it took the model to train in our cloud environment but also to ensure we got the same recall numbers as the research team did in their stack. Once that was accomplished, we decided the next goal should be to re-evaluate the model on a fresh dataset.

This fresh dataset was more representative of usage for our subscribers in the current timeframe. The model training logic accepted files that were a strange combination of comma, pipe and tab delimited records. We wrote a job that read in the current fresh dataset that consisted of Avro records and converted those Avro records to this strange format, instead of modifying the model training logic to read Avro‬.

Then a few days later, we realized the format for the dataset for training had to be one large file...

With this realization, we then decided to rewrite the model training logic so that the input for training wasn’t that strange format‬.

We then spent days on determining why the evaluation numbers (i.e. recall) were lower for the model trained on the old dataset vs the fresh / current dataset.

We eventually pivoted away from this deep learning model and opted for focusing on introducing a For You personalized homepage in our video product using an older collaborative filtering algorithm that was already available in our production environment.

During those months, maybe time would have been better spent working on the deficiencies in the data pipeline that resulted in opting for the old poorly formatted 2013 dataset initially (so that our research peers always had a fresh dataset to work with)...or maybe time would have been better spent going straight to production with this collaborative filtering algorithm and then A/B Testing that against deep learning models in the future.

Back in 2018, it felt like months were wasted on trying to get this deep learning model trained on a fresh dataset that was more representative of production.

Now, it still feels like months were wasted. Our engineering team was small, so focusing on the right work was important.

Disclaimer: I no longer work on video recommendations, so for all i know, this deep learning model could be serving millions of requests today.
